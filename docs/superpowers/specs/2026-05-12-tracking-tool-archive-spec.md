# 快递单号管理工具 — 半年归档方案

## 目标

解决长期使用（每日 500+ 单）的数据膨胀问题：存储容量、渲染性能、数据安全。通过半年周期归档 + 临时导入查看机制，让工具能无限期使用。

## 当前瓶颈

| 问题 | 细节 |
|------|------|
| localStorage 5MB 上限 | 约 40 天写满 |
| 全量渲染卡顿 | 500+ 卡片时 DOM 操作明显卡顿 |
| 无自动备份 | 手机丢/浏览器清数据则全丢 |

## 架构变更

**存储升级：localStorage → IndexedDB**

IndexedDB 优势：
- 容量大（可用磁盘的 50%，约 50-250MB）
- 异步操作（不阻塞 UI 线程）
- 支持游标遍历和分页查询
- 现代手机浏览器（含微信内置浏览器）全部支持

**渲染变更：新增分页**

每页渲染 50 条卡片，滚动到底自动加载下一页。搜索也在已加载范围内搜索，或追加搜索所有数据但限制返回最多 100 条。

## 接口设计

### 新增功能

| 功能 | 位置 |
|------|------|
| 归档 | 📋 FAB 菜单 |
| 临时查看 CSV | 📋 FAB 菜单 |
| 存储统计 | 列表页统计栏 |

### 归档流程

1. 用户点击「归档」
2. 显示确认弹窗：「当前 XXXX 条记录，导出后将清空数据开始新的半年」
3. 确认 → 导出 CSV（文件名格式：`2026上半年_快递记录_2026-06-30.csv`）
4. 导出完成后提示：「导出成功！已保存至下载目录」
5. 自动清空 IndexedDB，更新 `lastArchiveDate`
6. 回到空列表

### 临时查看流程

1. 用户点击「临时查看」
2. 文件选择器打开，选择 CSV 文件
3. CSV 解析后加载到内存变量 `importedViewData`
4. 页面顶部显示黄色提示条：「📄 查看历史数据 · 共 X 条」+「返回当前数据」按钮
5. 渲染引擎切换到 `importedViewData`
6. 搜索、Tab 切换均工作，但标记已收/删除等编辑操作禁用
7. 点击「返回当前数据」→ 恢复 `appData` 渲染

### 存储统计

统计栏增加：
```
共 3,240 条 · 已收 1,200 · 待收 2,040  |  上次归档: 2026-06-30
```

存储接近上限（>80%）时显示警告颜色。

## 数据层变更

**新增 IndexedDB 包装器：**

```javascript
function openDB() — 打开/创建 IndexedDB 数据库（db name: 'tracking_tool'）
function getAllRecords() — 读取全部记录
function addRecordToDB(record) — 单条写入
function addManyToDB(records) — 批量写入（用于导入）
function clearAllFromDB() — 清空全部（归档后用）
function getRecordCount() — 获取记录数（用于统计）
```

**保留 localStorage 用于元数据：**
- `lastArchiveDate` — 上次归档日期
- `lastCustomer` / `lastRegion` — 自动记忆输入
