# 快递单号管理工具 V2 设计文档

## 目标

优化移动端快递单号管理工具的录入效率和核对体验，解决 V1 的四个核心痛点。

## 当前状态（V1）

- 单文件 HTML（~1052 行），移动端 Web 应用
- 卡片式列表，三屏切换（列表 / 添加表单 / 详情）
- 客户+地区+单号三者皆必填，每次需重新输入
- 所有卡片混在一个列表，已收和未收不分
- 标记已收需点卡片→详情页→点按钮（3 步）
- 已部署至 GitHub Pages: `https://7ocbefore.github.io/tracking-tool/`

---

## 用户需求（4 项）

### 1. 已收/未收分离 — 顶部 Tab

卡片列表顶部新增两个 Tab：「待收件」（默认）和「已收到」，各自独立展示。

- 每个 Tab 显示该分类下的卡片数量
- 搜索范围跟随当前 Tab
- 空状态提示区分："暂无待收件快递" / "暂无已收件快递"

### 2. 快速标记已收 — 卡片上直接点勾

在待收件 Tab 下每张卡片右上角显示绿色勾选框 ☑。

- 点击 ☑ → 切换 `received` 状态 → 卡片从待收件列表动画消失
- 点击卡片主体 → 仍打开详情页（与 V1 一致）
- 勾选后结果即时反映在 Tab 统计数和已收件列表中

### 3. 验证规则调整 + 自动记忆

**验证规则：**
- 快递单号：必填
- 客户名称和地区：至少填一个（两个都不填不可保存）

**自动记忆：**
- 打开添加表单时，自动填入上一次保存的客户名称和地区
- 输入框下方显示最近 3 个「客户/地区」组合，点一下快速填充
- 用户可手动修改或清空

### 4. 长按批量标记

长按任意卡片进入批量模式：
- 每张待收件卡片左侧出现复选框
- 顶部出现操作栏：「已选 N 项」+「标记已收」按钮 + 「取消」
- 勾选多张卡片后点「标记已收」一键处理
- 点「取消」或再次长按退出批量模式

## 额外优化（5 项）

### 5. 撤销删除

删除操作后底部出现 toast：「已删除 [单号]」+ 「撤销」按钮，持续 3 秒后自动消失。
- 点「撤销」恢复记录
- 3 秒超时后执行真正的 `deleteRecord`
- 适用于详情页删除和批量删除

### 6. 导入/导出常驻按钮

新增灰色 FAB（📋），点击展开菜单：
- **导出 CSV**：下载所有记录为 CSV 文件（UTF-8 BOM）
- **导入 CSV**：从手机文件选择器读取 CSV，合并到当前数据（相同单号跳过）
- 作为误删恢复的兜底手段

### 7. 页面过渡动画

屏幕切换使用 CSS transition：
- 列表 → 添加表单：从右滑入（250ms）
- 列表 → 详情页：从右滑入（250ms）
- 返回列表：从左滑出（250ms）
- 采用 `transform: translateX` 实现，不依赖 JS 动画库

### 8. 文字 + 颜色双重状态标识

卡片左侧竖条颜色 + 文字标签组合：
- 待收件：橙色竖条（`#ffa940`）+ 右上角"待"标签
- 已收到：绿色竖条（`#73d13d`）+ 右上角"收"标签
- 确保色盲用户也能区分

### 9. 无结果搜索引导

搜索无结果时，空状态提示变为：
- "没有匹配的记录"
- 下方显示匹配到的搜索关键词
- 「添加此单号？」快捷按钮（点后跳转添加表单，单号已预填）

## 架构

**不变：**
- 仍为单文件 HTML，零依赖
- localStorage 持久化（key: `tracking_data_v2`，新增 schema 版本号）
- 三屏架构扩展为四屏：列表（含 Tab）/ 添加表单 / 详情 / 导入导出面板

**变化：**
- 数据 key 改为 `tracking_data_v2`，新增 `schemaVersion` 字段
- `render()` 拆分为 `renderPending()` 和 `renderReceived()`，根据当前 Tab 调用
- 新增 `softDelete(id)` + `undoDelete(id)` 函数支持撤销删除
- 新增 `importCSV()` 函数
- CSS 新增 Tab、过渡动画、批量模式、toast 等样式
- `addRecord` 调用后更新 localStorage 中的 `lastCustomer` 和 `lastRegion`
- form 打开时预填 `lastCustomer` / `lastRegion`

## 数据模型

```javascript
// V2 record shape (adds schemaVersion)
{
  id: "lx8k3m...",
  number: "SF1234567890",
  customer: "张三",       // 可选
  region: "北京",          // 可选
  notes: "",
  date: "2026/05/12",
  received: false,
  receivedAt: null,
  createdAt: 1715500000000,
  schemaVersion: 2
}

// localStorage keys
// tracking_data_v2:  主数据 (appData)
// tracking_last_input: { customer, region } — 上次输入记忆
```

**验证规则伪代码：**
```
if (!number) → 提示"请填写快递单号"
if (!customer && !region) → 提示"客户名称和地区至少填写一个"
→ 保存
```
