/ 核心界面

扫码匹配 & 自动标记

摄像头一扫，自动匹配并标记已收到。

✕ 扫码匹配 ✓ 2单
✓ 已匹配 — 已自动标记已收到

客户: 张三 地区: 上海

扫码匹配流程

  • 实时检测 — 调用 BarcodeDetector API，自动识别条形码
  • 模糊匹配 — 支持扫码内容与单号互相包含的模糊匹配逻辑
  • 自动标记 — 匹配成功后自动将状态更新为「已收到」并记录签收时间
  • 连续扫码 — 匹配后不清除扫描器，继续扫描下一个条码
  • 视觉&触觉反馈 — 屏幕闪光 + 振动提示，匹配结果卡片展示完整信息
  • 闪光灯支持 — 低光环境可开启闪光灯辅助拍摄

/ 数据流程

完整数据生命周期

从录入到归档的全链路闭环。

01

📥 数据录入

手动添加单条记录 / CSV 文件批量导入。支持从搜索框快捷跳转到添加页面。

02

📋 日常管理

待收件/已收到双 Tab 分离。搜索、排序、分页、批量标记 — 所有操作即时响应。

03

🔗 扫码匹配

收到包裹后拍摄面单条形码，系统自动模糊匹配待收件列表并标记已收到。

04

📤 导出归档

按季度/半年的周期一键导出 CSV 并清空数据库，开始新的数据周期。

📱

移动端优先

Safe Area 适配刘海屏/挖孔屏。触摸友好的交互尺寸，底部安全区域适配。所有操作可单手完成。

⚡

离线 PWA

基于 IndexedDB 本地存储，无需网络连接。Service Worker 缓存资源，可添加到手机桌面像原生 App 一样使用。

🎨

精致交互

卡片按压缩放动画、Toast 消息、模态框弹窗、无限滚动加载。使用 CSS 动画而非 JavaScript 驱动，性能更优。
